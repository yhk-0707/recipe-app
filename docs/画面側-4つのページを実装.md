# 1.app/page.tsx（検索トップ、旧 index.html）
- Reactは`index.html`だけ必要で、あとはJSXでOK  
→　一般的に SPA（Single Page Application） と呼ばれる仕組み
```
<!DOCTYPE html>
<html>
  <head><title>マイアプリ</title></head>
  <body>
    <div id="root"></div> 
    
    <script src="bundle.js"></script>
  </body>
</html>
```
`id="root"` という空っぽの器に対して、React（JSX）が「JavaScriptの力で、ボタンやテキストなどの部品（コンポーネント）をリアルタイムにはめ込んでいく」という動き

## htmlがindex.htmlの1つで済むと何が良い？
### 画面遷移が爆速になる  
ページを切り替えるたびに、重たいHTML全体をサーバーからダウンロードし直す必要がありません。  
必要な部分のデータ（JSX）だけをサッと書き換えるため、スマホアプリのように一瞬で画面が切り替わります。

### 部品（コンポーネント）の再利用がラク
例えば「共通のヘッダー」や「同じデザインのボタン」を、HTMLのコピペではなく、JavaScriptの関数として1回定義すればどこでも使い回せます。

## そしてNext.jsでは……
### Next.jsになると、この「1枚の土台」すら自動で管理
Next.jsは「サーバーサイド（裏側）で自動的にHTMLを組み立ててからブラウザに送る仕組み（SSR / SSG）」を持っているから

---

1. 素のReactの場合（クライアントサイド・レンダリング：CSR）
ブラウザに「空っぽの `index.html`」をまず送る。

    そのあと、ブラウザ側（ユーザーのパソコン）でJavaScriptを実行して、HTMLを組み立てる。

    だから： 土台となる `index.html` がどうしても1枚必要だった。
    
---

2. Next.jsの場合（サーバーサイド・レンダリング：SSRなど）
ユーザーからアクセスが来たら、Next.jsのサーバー（裏側）が `.tsx（JSX）`を読み込む。

    サーバー側で `<html>` や `<head>`、`<body>` タグを含んだ完全なHTMLをその場で自動生成する。

    ブラウザには、すでに中身がぎっしり詰まった完成品のHTMLが届く。

    だから： 固定の `index.html` ファイルをプロジェクト内に置いておく必要がそもそもない。

---

→　HTMLの `<head>`は`layout.tsx`が担っている。

# 2.app/recipes/page.tsx（登録済み一覧、旧 recipes.html）

- `src/recipes-list.js`
    - `initializeRecipes()`
    - `displayRecipeList(getRecipes())`

- `src/ui/ui.js`
    - `displayRecipeList()`
    - `renderRecipeListItem()`

---

- Next.js の `src/app/recipes/page.tsx` → 旧Reactの「一覧ページ本体」

- `prisma.recipe.findMany()` → 旧Reactの `initializeRecipes()` + `getRecipes()`

- `findRecipesByIngredientsOrName(...)` → 旧Reactの `searchRecipe()` 内の検索処理

- `<ul> ... </ul>`旧Reactの `displayRecipeList()` の描画部分

- `<Link href={/recipes/${recipe.id}}>`旧Reactの `detail.html?id=...` へのリンクに相当

補足：旧React版では、一覧の描画を `displayRecipeList()` に分けていた。今のNext.js版は `page.tsx` の中で直接描画している。  
なので、機能としては同じでも、構成は少しシンプル

# 3.app/recipes/[id]/page.tsx（詳細、旧 detail.html）

## React版で対応するもの
- `src/detail.js` → `getRecipeIdFromQuery()`
    → Next.js では `params.id`
- `initializeDetailPage()` → `page.tsx` の中`findUnique()`して描画
- `renderRecipeDetail(recipe)` → JSX での詳細表示部分
`getRecipeById(recipeId)` → `prisma.recipe.findUnique()`

---

### なぜ[id]というディレクトリに入れるの？
Next.js の App Router で「動的ルート」を作るため。

各レシピページのURL末尾は `/recipes/1`や `/recipes/123`のようになる。
この 変わる部分を Next.js に「ここは可変だよ」と伝える書き方が `[id]`

### なぜフォルダ名？
Next.js は ファイルとフォルダの構造そのものがURLになるから。
- `src/app/page.tsx` -> `/`
- `src/app/recipes/page.tsx` -> `/recipes`
- `src/app/recipes/[id]/page.tsx` -> `/recipes/任意の値`

★App Router（フォルダの階層でURL（画面）を決めるNext.jsのシステム）の書き方。
