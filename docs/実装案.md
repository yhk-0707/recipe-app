# レシピアプリ 各種実装案

## コード系
- `defaultrecipes`をモジュールとして切り出し

## 機能系
- 元アプリ同様のレシピ登録ページの実装
- 登録済みレシピページの実装
- 記述形式の異なる材料、手順の内容をAIを使って適切な形に整形
  - 処理の流れ
    1. レシピ全文を貼り付け
    2. React → FastAPIへ送信
    3. FastAPI → Gemini APIへ解析依頼
    4. AIが料理名・材料・手順をJSON形式で返却
    5. FastAPIでJSONを検証・整形
    6. Reactで編集画面を表示
    7. ユーザーが確認・修正
    8. PostgreSQLへ保存
    9. レシピ一覧へ反映


## アプリ本体系
- webアプリとして公開<small>（GitHub Pages → Vercelで試してみるとか）</small>

## 技術構成案
- フロントエンド：React + Vite<small>（ビルドツール）</small>
- UI公開：Vercel
- バックエンド：FastAPI <small>（API公開のためのフレームワーク（公開するなら経由する ※Reactで直接API実装するとAPIキーが漏れる））</small>
- バックエンド公開：Railway <small>（API公開用のサーバー）</small>
- AI：Gemini API
- データベース：PostgreSQL
- ソース管理：GitHub
