# ECS Fargate ハンズオン できたことまとめ

## 1. ネットワーク基盤を作成した
- VPCを作成した
- パブリックサブネットとプライベートサブネットを分けた
- ルートテーブルでIGWとNAT Gatewayを使い分けた

## 2. セキュリティ境界を分けた
- `alb-sg`、`app-sg`、`db-sg` を役割ごとに作成した
- SG参照で通信元を絞った
- インバウンドだけでなくアウトバウンドも通信経路に影響することを確認した

## 3. データベースを作成した
- RDSでPostgreSQLを作成した
- DB用のサブネットグループとセキュリティグループを設定した
- `psql` で接続確認した

## 4. 秘密情報を外出しした
- SSM Parameter Storeに接続文字列を保存した
- アプリの設定値をコードに直書きせず渡す形にした
- `GetParameter` と `GetParameters` の違いを確認した

## 5. コンテナをビルドして配布した
- Dockerイメージをローカルでビルドした
- ECRリポジトリにpushした
- アプリを配布可能なイメージとして扱った

## 6. ECSで動かした
- ECSクラスターを作成した
- タスク定義でコンテナの設計図を作成した
- 実行ロールとタスクロールを分けて設定した
- Fargateでコンテナを常駐実行する流れを確認した

## 7. 公開入口を作成した
- ALBを作成した
- ターゲットグループを作成した
- ECSサービスをALBに接続した
- URLでアクセスできるようにした

## 8. 障害を切り分けた
- ECSサービスのイベントを確認した
- CloudWatch Logsでエラーを確認した
- CloudTrailでAWS APIの失敗を確認した
- ネットワーク、IAM、DBスキーマのどこで壊れているかを切り分けた

## 9. 運用コストを意識した
- どのリソースがアクセス数で増えやすいかを確認した
- ALB、NAT Gateway、RDS、Fargateが課金ポイントだと確認した
- 使い終わったら片付けが必要だと確認した
