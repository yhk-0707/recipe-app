# TypeScriptファイルの拡張子まとめ

### tsファイル（TypeScript）
* **役割**
  画面の見た目を含まない、純粋なプログラムの処理や計算だけを書くファイルです。
* **特徴**
  HTMLタグ（JSX）を書くことはできません。書くとエラーになります。
* **主な使い所**
  データの計算、APIとの通信処理、Prismaの接続設定など。

### tsxファイル（TypeScript + X）
* **役割**
  画面の見た目（コンポーネント）を作るためのファイルです。
* **特徴**
  HTMLによく似たJSXという記述をの中に書くことができます。
* **主な使い所**
  Next.jsの画面（`page.tsx`）や、ボタン、ヘッダーなどのUIパーツ。

### d.tsファイル（型定義ファイル）
* **役割**
  具体的な処理は一切書かず、型（ルール）の情報だけを専門に書くファイルです。
* **主な使い所**
  JavaScriptで作られた外部のライブラリをTypeScriptで読み込む際に、エラーが出ないよう型を教えてあげるために使います。

### config.tsファイル（設定ファイル）
* **役割**
  開発ツールやフレームワークの設定をTypeScriptで書くためのファイルです。
* **主な使い所**
  Tailwind CSSの設定（`tailwind.config.ts`）など。入力補完が効くため設定ミスを防げます。

### test.ts / spec.tsファイル（テストコード用ファイル）
* **役割**
  作ったプログラムが正しく動くかを自動でチェック（テスト）するためのプログラムを書くファイルです。
* **主な使い所**
  テスト用のツール（Jestなど）を使う際に作成し、コードの動作検証を行います。

---

## 迷ったときのシンプルな基準

* 1行でもHTMLタグが登場するなら **`tsx`**
* 計算やデータ処理だけでHTMLタグが一切登場しないなら **`ts`**
