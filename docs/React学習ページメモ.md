# React学習ページメモ - 内容で学んだこと

## 01 JSX — HTML文字列の組み立てをやめる
- タグの閉じ忘れをエラーで教えてくれるの便利
- 文字列流し込みじゃなくてタグのように使える
- htmlの書き方は少し変わる（class→classNameになるとか、リンクタグとか？)

## 02 state — DOMを触るのをやめて、データを触る
- JSでは変数とDOMの両方を自分で更新、ReactはuseStateのデータのみ更新で画面はReactが合わせてくれるので、コードが短い

## 03 リスト表示とkey — 「全部作り直す」から「差分だけ」へ
- 「全部作り直し」と「差分だけ」のイメージが付きやすいデモが神！
- JavaScriptのindexは順番（配列）、Reactのkeyは名札。ピンポイントで管理してるReactの方がわかりやすい。

## 04 条件付きレンダリング — 「隠す」から「無くす」へ
- HTMLはあるけど隠してる、Reactは条件式であるなし切り替えするので、メモリ削減。

## 05 フォーム — 値を「取りに行く」から「常に持っている」へ
- 制御コンポーネントでは入力値が常にState（今回は入力フォーム）にあるので、リアルタイム更新
- Stateってつまるところ変数？（ただしReactが画面も更新してくれるのが違い）
- JavaScriptにも「変数で共通化」の考えはあるけれど、ReactのStateは画面更新と連動している

## 06 useEffect — localStorage保存を「呼び忘れない」仕組みに
- JavaScriptは何をきっかけに処理を実行するかで、ReactのuseEffectはどの状態(state)が変わったら処理を実行するか

## 07 コンポーネントとprops — 画面を「部品」で組み立てる
- コンポーネント=関数、props=引数

## 08 総合演習 — 材料検索をReactで書き直すと
- リアルタイム入力でシームレスな検索が可能  
　→あえて検索ボタンを置きたい場合は？  
　→→入力中のStateと、検索状態のStateを分けて作ればOK
