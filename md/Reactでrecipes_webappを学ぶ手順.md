## 1. まずは「3つの役割」を知ろう

Reactを始める前に、まずはHTML・CSS・JavaScriptの違いをざっくり理解しましょう。

- HTML: 画面の骨組み
  - 例: 文字、見出し、入力欄
- CSS: 画面の見た目
  - 例: 色、余白、サイズ
- JavaScript: 画面の動き
  - 例: ボタンを押したら何かが起きる

### たとえると

- HTML = 家の骨組み
- CSS = 家の内装や色
- JavaScript = 電気やドアの開閉

Reactは、これらをもっと扱いやすくする道具です。

---

## 2. Reactは何をしてくれるのか

Reactは、画面を「部品」に分けて管理しやすくする技術です。

たとえば、このアプリなら次のように分けられます。

- App: 全体の画面
- SearchBox: 検索入力とボタン
- RecipeList: 検索結果の表示
- RecipeForm: レシピ登録フォーム

### すごいポイント

「画面の一部が変わったら、その部分だけを更新しやすい」です。

これは、実務でもよく使われる考え方です。

---

## 3. まずは1つだけ学ぼう

最初は「状態」を学びます。

### 状態とは

状態とは、画面に表示する値が変わるものです。

たとえば:
- 入力欄に打った文字
- 検索結果の一覧
- フォームの中身

Pythonでも変数はありますが、Reactでは「その値が変わったら画面も自動で変わる」ことが大事です。

---

## 4. まずは小さな例から始める

完成形をいきなり見せるのではなく、まずはとても小さい形から始めます。

```jsx
function App() {
  // ここに入力文字を入れるための状態を置く
  // その値が変わると、画面も変わるようにする

  return (
    <div>
      {/* 入力欄 */}
      {/* 入力した内容を表示する場所 */}
    </div>
  );
}
```

この形が作れたら、次に次の順で進めます。

1. 入力欄を作る
2. 入力した文字を表示する
3. ボタンを押したら動くようにする

---

## 5. コードにはコメントを付ける

Reactを勉強するときは、コードにコメントを残すととても分かりやすくなります。

```jsx
function SearchBox() {
  // 入力欄の文字を覚えておくための場所
  // ここが変わると、画面も変わる

  return (
    <div>
      {/* ユーザーが入力する場所 */}
    </div>
  );
}
```

実務でも、コメントは「なぜこう書いたか」を残すためによく使います。

---

## 6. 実務ではどう書くか

実務では、次のように書くことが多いです。

- 1つの部品は1つの役割だけを持たせる
- 見た目と動きを分ける
- 変わる値は必要な場所にだけ持たせる
- 直接のDOM操作はあまりしない

### まず覚える言葉

- props: 親から子に値を渡す
- useState: 変わる値を管理する

難しく聞こえても、まずは「値を渡す」「値を覚えておく」という感覚で大丈夫です。

---

## 7. これからの学習順

1. まずは入力欄を作る
2. 入力した文字を表示する
3. 検索ボタンを追加する
4. 検索結果を表示する
5. レシピ登録フォームを作る
6. 既存の検索ロジックを React につなぐ

この順なら、初心者でもつまずきにくいです。
