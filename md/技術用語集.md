# 技術用語集

| 用語           | 一言でいうと                                            | メリット                             | デメリット                                       | つまづきやすい点                                 |
| -------------- | ------------------------------------------------------- | ------------------------------------ | ------------------------------------------------ | ------------------------------------------------ |
| HTML/CSS/JS    | ブラウザが直接読める                                    | 環境構築不要                         | 画面の複雑化に比例してコードが煩雑化             | 状態変化に合わせた画面更新を自前で書く必要がある |
| React          | UIを部品化して組み立てるJSライブラリ                    | 部品の再利用、状態管理が楽になる     | ルーティングやサーバー処理は別途必要             | JSX記法、useState/useEffectなど                  |
| Next.js        | Reactにルーティング・サーバー機能を足したフレームワーク | ルーティング・サーバー処理が標準装備 | Reactの基礎がないとNextとの境界が分かりにくい    | サーバー/クライアントコンポーネントの区別        |
| Laravel        | PHPのフルスタックWebフレームワーク                      | DB操作・画面表示まで一式揃っている   | PHP自体に慣れが必要                              | MVCの役割分担                                    |
| ECS on Fargate | AWS上でDockerコンテナを動かし管理するサービス           | スケーリングや再起動を自動化できる   | 覚える概念が一段増える（タスク定義・サービス等） | フロントエンド/バックエンドで異なるアプリの運用  |

進め方の比較・選択肢は`学習計画_概要版.md`を参照。
